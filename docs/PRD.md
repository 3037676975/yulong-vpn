# 玉龙VPN 2.0 PRD 产品需求文档

## 1. 项目背景

玉龙VPN 2.0 来自一个早期浏览器代理插件项目。旧项目里存在命名混乱、版本混乱、插件和 APK 混用、后台地址不统一、文档缺失等问题。

本次 2.0 的目标是把项目整理成一个可以开源、可以交接、可以继续开发的版本。

## 2. 产品目标

用一句话说明：

> 玉龙VPN 2.0 是一个带管理后台的 Chrome 插件配置与节点管理系统。

主要目标：

1. 插件端使用统一的黑金 UI 母版：v18-backend-check。
2. 后台可以管理节点、验证码、通知和统计。
3. 插件从后台读取配置，不再把关键配置写死在插件里。
4. 后台账号密码可以在登录后修改。
5. 项目文档足够清楚，小白也能部署。
6. 清理旧版本，避免 FanVPN、Android APK、临时包继续混淆。

## 3. 用户角色

### 3.1 管理员

管理员打开后台，可以完成：

- 登录后台
- 修改后台账号密码
- 设置动态验证码
- 管理节点
- 检测节点是否可用
- 查看使用统计
- 发布前台通知

### 3.2 插件用户

插件用户安装 Chrome 插件后，可以完成：

- 输入动态验证码
- 刷新节点
- 选择节点
- 连接或断开
- 触发真实测速
- 查看通知

## 4. 功能需求

### 4.1 后台登录

后台必须先登录才能使用管理功能。

登录来源优先级：

1. Supabase 中保存的后台账号设置
2. Vercel 环境变量中的 ADMIN_EMAIL 和 ADMIN_PASSWORD

### 4.2 账号密码设置

账号密码设置必须放在后台内部，不单独暴露成独立页面。

位置：

```txt
后台左侧菜单 -> 账号密码设置
```

要求：

- 登录后才能打开
- 弹窗或内嵌面板展示
- 可以修改后台邮箱
- 可以修改后台登录口令
- 保存后重新登录生效

### 4.3 动态验证码

后台可以设置 6 位动态验证码。

用途：

- 插件启动后要求输入验证码
- 验证通过后才能使用连接功能

要求：

- 支持自定义验证码
- 支持过期时间
- 后台首页可以展示当前验证码
- 插件调用 `/api/access-code` 校验

### 4.4 节点管理

后台可以管理代理节点。

节点字段：

- id
- name
- region
- server
- port
- scheme
- priority
- status

### 4.5 配置发布

插件通过接口读取配置：

```txt
/api/config
```

返回内容包括：

- 产品名
- 版本
- 是否需要验证码
- 节点列表
- 通知接口
- 统计接口

### 4.6 节点检测

后台支持检测节点是否可用。

接口包括：

```txt
/api/node-test
/api/node-test-all
/api/plugin-node-test-all
```

v18-backend-check 插件的“真实测速”走后台检测逻辑，不再以临时 Google 检测版作为最终母版。

### 4.7 使用统计

插件可以匿名上报使用事件。

事件示例：

- open
- verify_success
- verify_failed
- connect
- disconnect
- speed_test

后台可以统计：

- 总用户数
- 今日活跃
- 连接次数
- 测速次数
- 验证成功次数
- 验证失败次数

## 5. 非功能需求

### 5.1 安全

- Supabase service_role key 只能存在服务端环境变量。
- 不能写进浏览器插件。
- 不能提交 `.env.local`。
- 后台账号密码不能做成公开页面修改。
- 插件端不放私钥。

### 5.2 易用性

- 小白可以按照文档部署。
- 后台 UI 保持黑金风格。
- 版本命名统一。
- 文件目录要清楚。

### 5.3 可维护性

- 插件母版固定为 v18-backend-check。
- 后台 API 尽量保持向后兼容。
- 旧版本不要混进主线。

## 6. 项目范围

### 6.1 本期包含

- Chrome 插件正式母版
- Next.js 后台
- Supabase 数据库
- Vercel 部署
- 中文文档

### 6.2 本期不包含

- Android APK 正式交付
- Cordova 版本
- iOS 客户端
- 商业支付系统
- 大规模用户权限系统

## 7. 版本原则

正式版本命名：

```txt
玉龙VPN 2.0 / v18-backend-check
```

以后更新只在此母版上继续，不再混用：

- FanVPN 老包
- v18 google-check 旧检测版
- Android APK WebView UI
- 临时金色验证码版

## 8. 成功标准

项目整理完成后，应达到：

1. 新人打开 README 能理解项目是什么。
2. 小白按照部署文档能跑起来后台。
3. 管理员能登录并修改账号密码。
4. 插件能读取 api2 后台配置。
5. GitHub 仓库没有明显旧版本干扰。
6. 文档能说明这次踩坑和经验。
